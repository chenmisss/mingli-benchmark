# 研究三协议（草案 v0.9）：命理方法论效应的盲法受控试验

**状态**：设计稿。锁定后须外部时间戳（公开 gist 哈希 / OSF）方可开跑。前两项研究降级为"探索性先导研究"援引。

## 0. 与前研究的根本区别

| 维度 | 研究一/二（先导） | 研究三（本协议） |
|---|---|---|
| 提示词来源 | 研究者手写 + 生产资产混用 | 对称生成程序（见 §3），冻结后不可改 |
| 分析师 | 知臂身份 | **盲法**：臂编码，统计锁定后揭盲 |
| 污染控制 | 无 | 前置探测 + 主终点迁移至私有集 |
| 采样 | 单次 temp0.3 | 3 种子 × temp 0，provider 锁定并记录 |
| 统计 | 题级独立假设（有偏） | 命主级聚类自助法为主检验；等价性用 TOST |
| 预注册 | 同仓库当日 | 外部时间戳，锁定先于任何数据接触 |
| 生产资产 | 盲派提示词/K线注入 | **一律排除**（本研究与产品完全解耦） |

## 1. 研究问题（因果宣称的边界）

RQ1：在固定底模与固定信息基座下，来自六种命理体系的**规范文本转写提示词**对客观命题判断准确率的因果效应。
RQ2：任一体系相对"仅排盘"对照的效应是否 >0（优效）或落于 ±5pp（等价，TOST 界）。
明示不宣称：流派理论上限的比较（受转写程序约束）、跨底模普适性（单底模研究）。

## 2. 污染控制（先决条件）

- **前置探测**（本协议附录 A，先于一切正式运行）：
  P1 无选项自由作答命中率 vs 选项基率；P2 答案键直接回忆探针（"contest8_20XX 第N命主Q1答案字母"）；P3 选项顺序打乱敏感性（正确率随原始字母序显著变化 = 记忆信号）。
  判据：任一探针阳性（超随机 z>3）→ BaziQA 全体降级为"污染嫌疑集"，仅作次要终点。
- **主终点数据集**：新建私有集 D-priv——目标 ≥100 命主 × 5 题（见 §6 功效），来源为平台真实用户中**书面同意+匿名化**的命例（出生数据+由本人确认的客观事实），或团队线下征集；**答案永不公开**，仅发布哈希承诺（每题答案的 salted hash 于跑数前公示，揭盲后公布 salt 供验证）。
- 公开集（BaziQA 205 题）保留为次要终点与跨研究桥接。

## 3. 方法论臂的构成定义：转写提示词 + 体系排盘程序（缺一不可）

**原则**：大模型无法自行完成历法换算与体系排盘（四柱/大运/紫微十二宫/大限/实律数/条文/骨重），故每个方法论臂由两部分构成，作为整体接受检验：
- **计算件**：该体系的确定性排盘程序，冻结版本并记录哈希；
- **文本件**：该体系的规范文本转写提示词（生成程序见 §3.1）。

| 体系臂 | 计算件（引擎） | 引擎输出 |
|---|---|---|
| 子平/盲派 | 四柱排盘引擎（lunar-javascript 系） | 四柱+十神+藏干+**大运序列** |
| 紫微 | iztro 全盘引擎 | 十二宫星情+四化+**大限序列** |
| 命律 | 实律数引擎 | 五行实律数+党争比+机算判决 |
| 铁板 | 起数引擎（python） | 先天/五音/日时命数+相关条文 |
| 称骨 | 骨重折算器 | 骨重+档次 |
| 贝叶斯 | 队列基率表 | 人口统计先验 |
| A1 对照 | 四柱排盘引擎 | 同上（**无文本件**） |
| A0 对照 | 无 | 无 |

**引擎对齐规则**：①各引擎输出模板冻结、版本哈希公示；②逐臂披露注入内容的 token 长度（信息量差异作为已声明协变量报告，不假装可完全对齐——不同体系的盘本来就不一样大）；③引擎故障按命主记缺失并审计，不静默计0。
**效应归属**：四柱系诸派共享同一计算件，故其臂间差异可归属于文本件；紫微/铁板/称骨的臂效应为"提示词+引擎"捆绑效应，解释时明示不拆分。

### 3.1 文本件的对称转写程序（去作者效应）

- 每个体系选定 1–2 部**社区公认规范文本**（子平：《渊海子平》《子平真诠》；盲派：段氏公开讲义；紫微：《紫微斗数全书》；命律：原书§5；铁板：条文总纲；称骨：称骨歌原文）。
- 由**独立转写模型**（与作答底模不同家族，如 claude-opus-4.8）按统一模板将每个体系转写为 800±100 字方法论提示词：模板固定为"判断原则×6 + 断事对应表 + 决策规则"，转写指令全体系逐字相同。
- 每体系生成 **2 个独立转写版本**（不同种子）作为提示词敏感性重复；转写完成即冻结入库，哈希公示。
- 对照臂：A0 裸、A1 仅排盘（提示词为空/一句话，同前）。

## 4. 盲法实施

- **臂编码**：运行脚本将臂名映射为随机码（K1…K14），映射表 sealed（加密文件，密钥由用户保管）；结果文件仅含码名。
- **分析师盲**：统计分析（聚类自助、TOST、Holm）在码名上完成并输出锁定报告；随后揭盲替换标签。分析代码在跑数前冻结。
- **评分盲**：全自动解析（前置冻结的解析器），无人工改判；解析失败按预定协议处理（该命主该种子作废并记录，缺失≥10% 的臂触发格式审计而非静默计0——修正研究二的信息性缺失问题）。
- 作答协议改为**逐题独立请求**（消除同命主 5 题共补全的叙事相关；成本×5 可接受）。

## 5. 运行与采样

temp=0；每臂 × 每题 × **3 种子**（不同 seed 参数或3次独立请求，多数票为该臂答案，票分歧记录）；OpenRouter `provider.order` 锁定单一供应商并记录返回的 provider 元数据；同日窗口内完成全部臂以控模型版本漂移。

## 6. 功效分析（决定 N，先于采集）

题内聚类设计效应 DE=1+(m−1)·ICC，取 m=5、ICC=0.15（先导数据估计）→ DE=1.6。检出两臂 8pp 差（40% vs 32%）、双侧 α=0.05、功效 80%：独立样本需 ~590 题/臂，聚类校正后 **~950 题 → 190 命主**；若仅检 10pp 差则 ~120 命主。据此 D-priv 目标定为 **150 命主 / 750 题**（可检 ~9pp），预算：14 臂 × 750 题 × 3 种子 × 逐题 ≈ 3.2 万次调用 ≈ $25–60（v4-pro 价位）。
等价性（TOST ±5pp）所需 N 更大，届时按实际 ICC 更新并披露。

## 7. 统计方案（冻结）

主检验：命主级聚类自助（B=10000）对各臂−A1 差值出 95%CI；家族（六体系×2转写=12 比较）Holm 校正。优效：CI 下界>0；等价：TOST 双单侧 ±5pp。次要：题型分层（预定分类器冻结）、公开集桥接、种子间一致性（Fleiss κ）。所有"打平/无差异"表述必须由 TOST 支持，禁止以 n.s. 冒充等价。

## 8. 角色分离

提示词转写=模型B（冻结）；运行=脚本（冻结）；答案密封=用户；盲态分析=研究者；揭盲见证=用户。利益声明：团队运营命理产品，本研究与产品资产解耦即为此利益冲突的缓解措施之一，仍如实声明。

## 9. 里程碑

M0 本协议外部时间戳 → M1 污染探测（阳性则改道D-priv-only）→ M2 提示词对称生成+冻结+哈希 → M3 D-priv 征集（150命主，唯一需要人力周期的环节）→ M4 全臂运行 → M5 盲态统计锁定 → M6 揭盲+论文。
D-priv 未就绪前，可先以公开集完成 M2/M4/M5 的"方法学彩排"（结果标注为污染嫌疑口径）。

## 附录 A：污染探测操作定义

20 命主 × 3 探针 × 3 次采样；P1 以自由文本与 gold 选项语义匹配（冻结的匹配规则）计命中，与 25% 基率比 z 检验；P2 直接问字母，与 25% 比；P3 同题两种选项排列各答 3 次，一致性差 >20pp 记阳性。任一探针 z>3 判污染阳性。
