# 神机妙算 · 命理 AI 公开评测台（Benchmark + MCP）设计草案 v0.1

状态：设计稿，待拍板后立项实现（测试端先行）。2026-07-11。

## 定位

sjms.ai 做"命理 AI 的考场"：我们出题、判分、发榜；第三方系统（青囊、各派专家系统、端到端大模型）经 MCP 接口应试。角色类比大模型评测：我们是 benchmark host，他们是被测系统。

## MCP 接口（server 端工具）

| 工具 | 入参 | 出参 | 说明 |
|---|---|---|---|
| `list_exam_sets` | — | 考集列表（公开集/私有集、题量、年份、规则版本） | 公开集可反复刷；私有集每系统限次 |
| `get_exam_paper` | set_id, team_token | 题目数组：命主生辰（年/月/日/时/性别/出生地）+ 四选一题面；**无答案** | 私有集出题顺序随机化＋每队独立乱序，防交换答案 |
| `submit_answers` | set_id, answers[], run_meta(模型/版本/是否联网) | 总分、分年切片、95%CI；私有集只返回分数不返回逐题对错 | 服务端判分，答案键不出库；私有集逐题对错不回传（防打点爆破） |
| `get_leaderboard` | set_id | 榜单（系统名/成绩/CI/提交时间/run_meta 摘要） | 公开集与私有集分榜 |
| `register_team` | 联系方式 | team_token | 实名制+频控 |

## "刨除记忆"三道闸（防训练数据污染）

1. **时间切分**：榜单主成绩只认被测系统声明的训练截止日**之后**公布的新题（每年大赛新卷 + 我们的滚动私有集）。
2. **私有题集**：经命主知情同意收录的真实命例（脱敏），答案键从未上网、永不发布；只回分数。冷启动可先用"已核验但未数字化传播"的纸质来源命例。
3. **背题探针**：对公开年份先跑"只给题不给盘"的裸测。裸测成绩显著高于随机（>35%）判定背题，该部分成绩降权并标注。

## 训练/评测分离

- 公开集 = dev：随便调参、不限次数、成绩不上主榜。
- 私有集 = holdout：每系统每版本一次提交；榜单只认它。
- 私有集按季度轮换补充，旧私有集自动转公开集（透明化+可复现）。

## 判分与统计

复用本仓库已冻结的分析口径：答出题准确率、按命主聚类的 bootstrap 95%CI、分年切片；榜单差异标注是否过 Holm 校正（防止把噪声当排名）。

## 防作弊面

- 私有集不回逐题对错、限次、每队独立乱序（已述）
- run_meta 强制申报是否联网检索（联网系统单列一档——历届答案可被搜到）
- 提交内容留档，可复核争议

## 与现有资产的关系

- 判分脚本：`scripts/study3-analyze-v2.ts` 口径直接复用
- 题库结构：BaziQA schema（subjects/questions/answer）
- 私有集载体：Firestore `benchmark_private_sets`（答案键单独收敛在服务端，不进任何客户端可达接口）
- 人类基线：`train-data/human-prize-tables.json` 逐年冠军带宽，榜单旁列作参照
