## 摘要

命理类 AI 产品常以“用户觉得准”作为有效性的证据，但这种主观反馈容易受到巴纳姆效应、确认偏差和叙事暗示影响。本文选择一个更窄、也更容易核验的问题：当输入出生信息与一组具有赛事官方答案键的选择题时，不同命理提示与工具管线能否让大语言模型稳定提高答对率？

我们使用 BaziQA 整理的 2021—2025 年公开赛事题目，对 14 个实验条件进行跨模型探索性评测：两个对照条件，以及六类命理方法各两份提示词版本。数据共 **200 题、41 名命主（题目所描述的人）**；同一命主包含多道题，因此统计单位不能简单视为 200 个相互独立样本。三个模型在同一批题目上作答，实验条件以 K 码遮蔽，条件身份在锁定分析输出之后才揭示。主要描述指标为**全集准确率**：解析失败、弃答和平票均计为错误。

在全集口径下，“仅提供程序化排盘、不提供方法论指导”的三模型描述均值为 **36.2%**；铁板-v2 为 **36.0%**。0.2 个百分点的差异不足以支持可靠排序。12 个方法管线的三模型描述均值均未超过仅排盘对照，但不同模型对实验条件的排序波动明显，因此这只能读作“本次实验没有观察到稳定正收益”，不能读作“证明各流派无效”或“六类方法彼此等价”。

另一次协议不同的十臂单模型实验中，包含仅排盘、裸生辰与“只按人口基率作答”三个相关条件。修正旧版 N=205 分母错误后，三者分别为 **85/200（42.5%）**、**84/200（42.0%）**和 **54/200（27.0%）**。命主级事后配对重分析显示，仅排盘与裸生辰差 **+0.5 个百分点**，区间跨零；人口基率提示控制比仅排盘低 **15.5 个百分点**。后一个结果说明这段提示策略不适合本题集，但不代表“统计学不如命理”，也不能单独证明题库存在反基率选择偏差。

本研究仍有公开题训练污染未知、闭源模型版本漂移、样本量有限、处理管线复合混淆以及执行偏差等限制。因此全部方法比较均按探索性证据解释。本文的主要贡献不是给命理贴上“科学”或“迷信”的标签，而是公开一套可审计的评测过程，并据此提出滚动私有题、满覆盖计分、命主级统计、污染控制和赛后揭榜等开放基准设计。当前实现已作为公网 Beta 参考台部署（bench.sjms.ai），但仅开放公开集、正式私有集仍为空，尚不能产生官方认证成绩。

## 一、本文测量什么，以及不测量什么

### 1.1 可回答的问题

本文试图回答四个有限问题：

1. 在同一批公开赛事选择题上，不同提示与工具管线的**全集准确率**是多少？
2. 某个方法管线的点估计优势，能否跨提示词版本和三个固定模型保持方向一致？
3. 一段人口基率提示控制，在这套赛事题上表现如何？
4. 在题目口径能够对齐的年份，所测 AI 管线与赛事冠军的观测分数相差多少？

### 1.2 本文不作的主张

- 本文**不证明命理学是科学，也不证明命理学无效**。回顾性赛事选择题表现不是出生信息与真实人生之间的因果证据。
- 本文**不证明六类方法彼此等价**。样本量只允许我们说“没有检出稳定增益”，不能说“任何条件下都不存在差异”。
- 本文**不把冠军分数视为普通命理师水平或人类能力上限**。冠军是参赛人群中的年度极端值，不同年份卷面难度也不同。
- 本文**不以 benchmark 分数证明产品在现实咨询中有用、安全或适合重大决策**。医疗、财务、法律、婚育、雇佣、保险等决定不应依据此类输出。
- 本文**不能完全排除训练数据污染**。公开题结果只作为探索性证据；正式榜单需要答案公布前的新题和受控私有评测。
- 对闭源 API，本文能够保证的是**归档材料可审计、既有结果可复算**，而不是未来重新调用时仍能逐字复现相同输出。

## 二、为什么需要客观评测，以及已有工作做到哪里

“用户觉得准”可以作为产品体验指标，却不足以单独证明客观事实判断准确。Forer 的经典课堂实验展示了笼统人格描述引发的个人验证效应；占星领域也已有双盲方法学先例。它们与八字并不是同一对象，但共同提醒我们：主观认同、客观事实命中和前瞻预测效度是三个不同问题，不能混为一谈。[1][2]

本文使用的 200 道题并非由我们首创。BaziQA 已将 2021—2025 年赛事题整理为标准化的大模型评测集，并报告模型在符号与时间组合题上的表现；目前也已经出现基于 BaziQA 的公开在线榜单。[3][4] 因此，本工作的创新定位不是“第一个命理 benchmark”，而是：

1. 对提示方法、工具管线和模型交互做遮蔽的探索性比较；
2. 整理带来源等级的人类冠军观测分；
3. 公开失败、偏差、逐题结果与复算材料；
4. 在 BaziQA 之上设计一个面向长期运营的评测主机原型，包括私有新题、提交治理、联网分轨与赛后揭榜。

LLM benchmark 的通行经验还要求多指标、明确缺口、污染控制和防止反复使用保留集。HELM、可复用保留集研究以及数据污染文献均说明：公开题上的一次高分不等于未见数据上的泛化能力，频繁查看私榜反馈也会让榜单本身成为训练信号。[5][6][7]

## 三、数据、金标与伦理边界

### 3.1 题目与命主

- **题库**：BaziQA 整理的全球算命师大赛 2021—2025 年题目。
- **规模**：200 道四选一题、41 名命主；随机猜测期望为 25%。
- **结构**：同一命主通常有多道题，涉及婚姻、财富、事业、六亲、健康与具体年份事件。
- **性质**：全部属于回顾性选择题；题目和答案已经进入互联网，训练污染风险不能排除。

公开题可以测试固定系统在固定题面上的表现，却不能在没有额外负对照的情况下证明模型使用了命盘信息。未来确认性研究至少需要加入：只给题目与选项、随机错配命盘、错误生日或伪盘、正确命盘四类条件；真正有解释力的是“正确命盘相对错配命盘的增量”。

### 3.2 金标状态

2022、2023、2024 年 BaziQA 金标答案已与主办方官方答案键逐题交叉核对；2025 年金标尚未完成官方逐题键的独立核验。人类成绩来自主办方可获得的官方得奖表、文字页与图片转录；2013—2015、2017、2021 年奖表缺失，2025 年成绩仍是两跳转述。[8] 转录数据见[官方答案键转录](train-data/official-answer-keys.json)与[人类奖表转录](train-data/human-prize-tables.json)。

这意味着：2025 年可以进入探索性复算，但不应作为正式私榜或强结论的唯一依据；任何金标修订都必须触发数据版本变更和结果重算。

### 3.3 隐私、版权与用途

出生时间、性别以及婚姻、疾病、死亡、财富等经历具有敏感性，即使不含姓名，也可能与其他公开信息组合后重新识别。当前本地参考实现不等于已经取得面向公网、第三方模型处理和长期存储的完整授权。正式部署前必须完成：

- 题面和赛事材料的使用授权或明确法律依据；
- 私有命例的书面知情同意、数据最小化、撤回与下架机制；
- 出生地等高识别度字段的模糊化；
- 数据保存期限、访问审计与泄露响应；
- 明确禁止把榜单分数包装成针对真人命运的科学认证。

## 四、实验设计与偏倚控制

### 4.1 十四个实验条件

实验包含两个对照：裸生辰，以及仅提供程序化排盘但不提供方法论指导；另有子平、盲派、紫微斗数、命律预言、铁板神数、称骨六类方法，各两份提示词版本。

必须强调：这里比较的是**复合管线**，不是纯粹的“流派文本”。各条件同时可能改变提示词、专属引擎、输入长度和信息结构。因此，观察到的差异不能唯一归因于某个流派理论。要识别方法论文本的因果效应，未来需要采用因子设计：统一事实底座 × 方法提示，并把专属引擎另行消融。

六类提示词由 Claude Opus 4.8 按统一模板生成两个版本，但除命律外，多数并非从逐字规范原文转写；12 份中有 5 份超过原长度约束，且 Opus 同时参与提示词生成和一个答题轨道。这些限制意味着“两份版本”只能用于观察提示敏感性，不能代表整个流派的理论上限，也没有完全消除作者效应。

### 4.2 三个固定模型，而不是三台独立测量仪

| 轨道 | 模型 | 重复口径 | 已知限制 |
|---|---|---|---|
| DeepSeek | `deepseek/deepseek-v4-pro` | 3 次响应，多数票 | OpenRouter 承载方未预先锁定 |
| Gemini | `google/gemini-3.1-pro-preview` | 分析统一取第一次响应 | 运行中由 Vertex 切换至 OpenRouter；部分文件因 bug 额外跑了 3 次 |
| Opus | `anthropic/claude-opus-4.8` | 分析统一取第一次响应 | 与提示词生成模型同源；部分文件额外跑了 3 次 |

三个模型可能共享训练语料，也采用不完全相同的重复协议，所以它们不是统计意义上的独立同分布样本。文中的“三模型均值”仅是对这三个固定系统的描述性汇总，不外推到所有大模型。

按最终纳入分析的响应计算，DeepSeek 为 8,400 次（14 条件 × 200 题 × 3 次），Gemini 和 Opus 各 2,800 次，共 14,000 次。附属轨种子 bug、失败、重试和废弃运行产生了额外调用，但没有形成完整的逐次计费账本。因此本版不再用“约 2.6 万次调用”作为规模卖点，更不把 API 调用数当作独立样本量；有效统计规模仍是 41 名命主、200 道题。

### 4.3 三重偏倚控制，而非临床“三盲”

本实验采用了三项程序性控制：

1. **会话隔离**：每题独立请求，避免同一会话内的前题答案和叙事影响后题。
2. **条件标签遮蔽**：结果以 K1—K14 保存，统计时不显示真实条件名。
3. **揭示前锁定**：真实条件身份在三轨锁定输出完成后才揭示。

这不等于临床试验意义上的三盲。运行者与分析者是同一研究者，映射文件曾以明文存在，分析器也在首批结果产生后因审计而重写。因此本研究的准确定位是：**协议引导、臂标签遮蔽的探索性公开题实验**。

### 4.4 主要执行偏差

| 偏差 | 对解释的影响 | 本版处理 |
|---|---|---|
| 污染探针未达到原协议规格 | 无法排除模型记忆公开题 | 全部绝对分按污染未知的探索性结果解读 |
| 提示词并非严格规范原文转写 | 条件不能代表流派理论上限 | 改称“提示与工具管线”，不作流派本体判断 |
| K 码映射明文、分析器晚于首批结果冻结 | 遮蔽强度低于原计划 | 删除“三盲”和确认性表述 |
| 排盘器注入 2026 当前流年、出生地即现居地等协议外字段 | 裸生辰与带盘条件比较受混淆 | 不用实验一证明排盘具有预测增益 |
| 运行器、实际推理服务商与种子参数在运行中修正 | 重复性和轨间可比性下降 | 逐文件保留服务商记录；三轨分开报告 |
| 原分析器 RNG、缺失和平票处理有缺陷 | 原始确认性分析不可继续使用 | 采用修正版逐题结果；本版主表改为全集描述重分析 |
| 当前 v2 配对 bootstrap 未复用同一命主抽样，TOST 实现也不标准 | 不能依赖原 pairwise CI 与等价性结论 | 本版不再引用“91 对无显著/无一等价”作为主证据 |

全部时间线和原始修正记录见 `docs/study3-amendment-1.md` 与 `docs/study3-protocol.md`。

### 4.5 指标与复算口径

- **主描述指标：全集准确率** = 正确题数 ÷ 200；解析失败、弃答和平票计错。
- **辅助指标：有效作答覆盖率（coverage）** = 获得有效答案的题数 ÷ 200。
- **条件准确率**只用于诊断，不用于排名，因为它会奖励选择性弃答。
- 所有不确定性分析以**命主**为聚类单位，而不是把同一命主的多题当成独立样本。
- 本版不宣称方法等价；等价结论必须来自预先设定界值和正确实施的双单侧检验。

复算脚本为 `scripts/benchmark-whitepaper-reanalysis.mjs`。

| 紫微-v1 | 31.5% | 37.5% | 37.5% | 35.5% | 81.5%–98.5% |
| 子平-v1 | 30.5% | 39.0% | 34.5% | 34.7% | 80.5%–99.0% |

| **裸生辰** | 30.5% | 29.0% | 39.5% | 33.0% | 78.0%–99.0% |
| 盲派-v1 | 28.0% | 35.5% | 35.5% | 33.0% | 79.0%–98.5% |

| 命律-v2 | 25.0% | 32.5% | 32.0% | 29.8% | 80.5%–98.5% |
| 命律-v1 | 24.0% | 32.5% | 32.5% | 29.7% | 75.5%–98.5% |

按命主成组重抽样的描述性敏感性分析中，仅排盘均值为 36.2%（95%区间 **[30.8%, 41.6%]**），铁板-v2 为 36.0%（**[32.0%, 40.0%]**）；配对差为 **+0.2 个百分点**（**[−3.8, +4.3] 个百分点**）。这一区间直接说明，当前数据不足以支持二者的可靠排序。

### 5.1 可以从表中读出的结果

1. “仅排盘”与铁板-v2 的描述均值只差 **0.2 个百分点**，不构成有意义的排名证据。
2. 在这组具体提示和引擎实现中，12 个方法管线的三模型描述均值没有超过仅排盘对照；但各模型的排序变化很大，没有出现清晰的跨模型冠军。
3. 同一方法的两个提示词版本可能相差数个百分点，说明提示实例本身是不可忽略的变异来源。
4. 命律两个版本在三模型描述均值上都较低，可作为后续验证的负向信号；在完成正确的确认性配对检验前，不称其为“确定负效应”。

### 5.2 这张表不能证明什么

它没有证明流派之间不存在差异，也没有证明排盘本身提供了现实预测信息。各条件是提示、工具和信息格式的复合处理，公开题污染又没有排除。当前最稳妥的事实表述是：**在本次 41 名命主、200 题和三个固定模型上，没有一个方法管线在三个模型中的点估计都高于仅排盘对照。**

## 六、实验二：只按人口基率作答，会发生什么？

这一节抽取自另一套十臂、单模型、按命主批量作答的实验，不是独立验证，也不与实验一的逐题独立协议直接混算。原文曾误把真实 200 题按 N=205 计算，并把全集准确率标成“答出题准确率”；本版已从归档结果重新复算。

| 管线 | 正确数 | 全集准确率 |
|---|---:|---:|
| 仅排盘 | 85/200 | **42.5%** |
| 裸生辰 | 84/200 | 42.0% |
| 人口基率提示控制 | 54/200 | **27.0%** |

按命主成组重抽样的事后点对点重分析（100,000 次、固定种子）得到：

- 仅排盘 − 裸生辰：**+0.5 个百分点**，按命主成组重抽样得到的 95% 区间为 **[−8.0, +8.9] 个百分点**，配对符号翻转 `p=1.000`；
- 仅排盘 − 人口基率提示控制：**+15.5 个百分点**，相应 95% 区间为 **[+3.5, +27.3] 个百分点**，未校正双侧 `p≈0.021`。

这些区间和 p 值均为审稿后新增的事后比较，没有按原十臂检验族做多重校正，因而只作探索性敏感性结果，不替代预先指定的确认性检验。

这支持两个有限结论。第一，本实验没有发现仅排盘相对裸生辰的准确率增量，因此不能用它证明排盘信息具有预测力；程序化排盘仍可能有减少历法计算错误的工程价值。第二，这段人口基率提示在本题集上表现较差。

这里测试的不是一个具有经验证先验、似然函数与校准数据的完整贝叶斯模型，因此不再称为“纯贝叶斯”或“纯统计学”。低分也不能单独证明出题人系统性选择了反基率命主。赛事偏好非典型案例是一种可能解释；其他解释还包括基率来源与地区、年代不匹配，提示执行失败，以及选项并不对应可比较的人口事件。验证“反基率选择偏差”需要逐题给出可信基率，并直接检验正确选项是否系统性具有更低先验概率。

## 七、人类参照：只能做有限的同卷描述比较

2018、2019、2020、2022、2023、2024 六个有资料的大题量年份，官方冠军原始分的未加权描述均值约 **48.9%**，范围为 **42.5%—65.0%**。这反映了卷面差异和极端值选择，不能称为稳定的“人类真实水平”。“已公布得奖表中最高”也不必然等于全体参赛者的最高 raw，因为现有材料未必覆盖未获奖者的逐题原始分。

### 7.1 为什么本版暂不报告“稳定的人机差距”

旧版曾列出 2022—2024 的 AI 年度点分，并据此宣称落后冠军 10—19 个百分点。复核后，本版撤回这一汇总：2023 年 AI 使用 40 题子集，而冠军分数来自完整 45 题；更重要的是，当前仓库没有为这组三年点分完整归档同一个 AI 配置的全部原始输出与不可变运行标识，无法证明它们来自预先指定、完全一致的系统版本，而不是开发过程中的一次配置。

未来人机表必须同时满足：完全相同题目子集、同一全集计分规则、预先锁定的 AI 版本、模型与实际推理服务商、运行日期、提示协议、原始输出文件，以及明确的人类比较对象（官方冠军或已公布得奖表中的最高 raw）。在这些条件满足前，人类成绩只作描述性背景，不计算“稳定差距”。

## 八、年份题探索：稳定发现的是“更难”，不是“找到了应期公式”

问题挖掘采用时间分半验证：前半为 2021—2023 年，后半为 2024—2025 年。分类规则很粗，只要题干或选项含具体公历年份，就记为“年份题”；其余记为“非年份题”。结果如下：

| 切片 | 年份题 | 非年份题 | 非年份题优势 |
|---|---:|---:|---:|
| 2021—2023 | 32.1%（n=42） | 40.3%（n=78） | +8.2 个百分点；95%区间 [−0.2, +16.6] |
| 2024—2025 | 21.7%（n=16） | 37.0%（n=64） | +15.3 个百分点；95%区间 [+4.8, +25.2] |

两个时期方向一致，支持“精确年份定位是当前系统的困难切片”。但针对流年冲合、日支关系等传统应期信号的训练—保留检验没有得到方向稳定的增益；样本量也不足以排除微弱效应。因此，现有数据支持的是**降低精确年份断言的置信度**，而不是给某个应期公式增加权重。方法、候选规则及反例见规律挖掘终局报告和分半验证数据。

作为补充，作者也把公开题用于自有产品链路的事后消融。该案例因反复接触同一题集而存在过拟合风险，移至[附录 A](#附录-a产品链路内部案例事后证据)，不作为主实验结论。

## 九、目前证据支持的工程结论

1. **确定性计算适合交给程序。** 程序化历法与排盘有可测试、可复核的工程价值；但本研究没有证明排盘信息增加了现实事件预测力。
2. **方法提示需要逐版本验证。** 当前没有一个方法管线在三个模型中的点估计都高于仅排盘；这不等于所有领域知识注入都无效。
3. **模型输出具有运行间变异。** DeepSeek 三次重复的逐题 Fleiss κ 在各臂约为 0.36—0.52；κ 越接近 1 代表一致性越高，这一区间只达到中等程度。它说明逐题答案并不稳定，但不能直接推出所有小于5个百分点的变化都是噪声。升级必须通过固定题集、多次重复和命主级区间检验。
4. **精确年份题是已观察到的困难切片。** 现有分半验证显示年份题在两个时期都更难；传统应期信号没有通过保留切片验证。更谨慎的产品策略是降低单点断言强度、邀请用户确认，而不是把噪声写成精确年份公式。
5. **公开旧题适合开发，不适合最终认证。** 反复调试后的高分应视为开发集成绩；真正的外部有效性必须来自未公开新题。

## 十、开放 benchmark：当前是参考实现，不是正式认证台

这项工作目前已经落成一套**参考实现**：统一题库、固定版本评分器、HTTP API，以及本地 stdio 形式的 MCP 适配层。它可以演示注册、领题、提交、判分和榜单流程；**现已作为公网 Beta 参考台部署于 [bench.sjms.ai](https://bench.sjms.ai)，但仅开放公开集、成绩非官方，还不是可对外认证成绩的正式评测台**。

### 10.1 当前三类考集

| 考集 | 内容 | 当前用途 | 是否产生官方排名 |
|---|---|---|---|
| `public_dev` | 2021—2023 公开题，金标答案可下载 | 开发、调参、复现 | 否 |
| `public_test` | 2024—2025 公开题；答案已进入互联网 | 自查与接口演示 | 否 |
| `private` | 答案从未公开的新命例 | 未来正式评测 | **当前为空，尚未开放** |

其中 2025 年金标尚未完成官方逐题交叉核验。隐藏 API 返回中的答案并不能让已经在互联网上公开的题重新变成私有题。

### 10.2 已实现的能力

- 题面去重与同命主强制同切分；
- 数据集完整内容哈希和版本锁定；
- 领题接口不下发金标答案；
- 按参赛应用确定性重排选项；
- 缺答计错、满覆盖才允许入榜；
- 提交配额、速率限制、API key 哈希与审计日志；
- HTTP 与 MCP 共用评分核心；
- 托管 endpoint 强制归入联网赛道，并有基础 HTTPS、私网地址与响应大小检查。

这些机制仍有边界。选项重排只能防止记忆答案字母，不能防止模型记住题目内容；训练截止日对闭源滚动模型难以核验；离线/联网状态仍部分依赖自报。

其中“私有榜探分侧信道”已在本次修复：`private` 集在揭晓前不再返回任何分数桶、排名或 `p_vs_top` / `p_vs_prev`，只回提交计数（见 10.3）。仍需注意：本地 stdio MCP 适配器会直接实例化评分服务并读取本地数据，因此未来私有金标绝不能随参赛者本地适配器分发，公网只提供连接受控远程服务的薄客户端。

### 10.3 上线状态与仍待完成项

本参考台已部署为公网 Beta（bench.sjms.ai），仅开放公开集、成绩非官方。相对审稿版，本次已解决以下阻塞项：

- ✅ **托管 endpoint 安全默认关闭**：默认禁用，需显式设置环境变量才能开启；线上以真 key 实测提交返回 403。
- ✅ **私榜零泄露**：`private` 集在揭晓前不返回任何分数桶、排名或 `p_vs_top` / `p_vs_prev`，只回提交计数；线上实测 `results_revealed:false`。
- ✅ **接口契约统一**：OpenAPI、HTTP 路由与示例统一为 `public_dev` / `public_test` / `private`，删除旧 `dev` / `holdout` 枚举。
- ✅ **榜单文本转义**：榜单渲染对参赛方名称与模型名做 HTML 转义，消除存储型 XSS。
- ✅ **服务端持久化**：提交、应用与揭晓状态改由 Firestore 持久化，跨实例一致（不再仅本地文件）。

升级为**正式认证台**仍需完成：

1. **非空私有集**（最关键）：征集答案从未公开的新命例，并取得允许评测、存储与处理敏感信息的明确知情同意。当前 `private` 仍为空，因此现阶段不产生任何官方成绩。
2. **组织级治理**：身份审核、每组织提交预算、固定评测窗口与统一赛后揭榜。（私榜比赛期零泄露已就绪，其余治理机制待建。）
3. **私有数据隔离**：公网 MCP 只作评测方远程 API 的薄客户端；private 题与金标只存在于评测方受控环境，绝不进入可分发的本地 MCP 包。（云端已服务端化；本地 stdio 适配器仅供公开集自测。）
4. **安全加固（其余项）**：内容安全策略（CSP）、跨实例持久化限速、正式密钥管理，以及独立第三方安全审计。
5. **统一运行条件**：明确联网、外部检索（RAG）与工具规则，冻结评分协议、数据清单、代码提交与运行环境。
6. **赛季与退役机制**：私题只在固定窗口使用；赛后转为公开开发集，并以新私题替换，避免保留集被长期自适应利用。

（统计口径——以全集准确率为主指标，报告 coverage、命主聚类区间与最小实际差异，`p≥0.05` 只表示证据不足、不作等价证明——已在评分器实现并持续作为报告规范。）

只有当非空私有集被封存、参赛条件可执行、数据治理与安全审计完成后，这套参考实现才会升级为正式公开 benchmark。在此之前，公开页面上的成绩一律标为“存在污染风险的开发或自查结果”，而不是科学认证。

## 十一、局限、利益冲突与现实用途边界

- **构念效度**：赛事回顾性选择题不等于现实前瞻预测，也不覆盖命理咨询的全部场景。
- **公开题污染**：训练数据可能包含题目、答案或相关命例；弱污染探针阴性不能证明无污染。
- **样本量**：只有41名命主；同命主多题相关，细小差异难以稳定区分。
- **复合处理**：提示词、专属引擎、输入长度和信息结构共同变化，不能把差异唯一归因于流派。
- **闭源模型漂移**：模型别名、供应商路由和服务端权重可能变化，重新调用不保证复现原输出。
- **金标与转录**：部分年份缺少官方逐题键或奖表；2025 年仍待核验。
- **人类参照**：冠军是年度极端值；缺少全体分布和逐题答案，不能估计稳定的人机总体差距。
- **作者利益冲突**：作者同时运营 sjms.ai、制定 benchmark 规则并持有评测基础设施。正式榜单需要独立治理、规则变更日志和第三方审计；作者自有产品不得接触私题或获得额外提交机会。
- **高风险用途**：任何分数都不得用于医疗、法律、财务、婚育、雇佣、保险或对真人作确定性命运断言。

## 十二、结论

在这套公开赛事选择题及当前三个固定模型上，没有一个方法提示与工具管线在三个模型中的点估计都高于“仅提供程序化排盘”的对照。不同提示版本与不同模型之间的排名波动说明，小幅领先很容易受到措辞、解析覆盖率、服务端变异与样本噪声影响。这个结果不等于“证明所有流派无效”；更准确的表述是：**在本研究覆盖的模型、提示、工具和公开题目中，尚无方法展示出可跨模型复现的准确率增益。**

程序化排盘仍有减少可验证计算错误的工程价值，但本研究没有证明盘面能够预测现实人生，也没有证明赛事答题能力能够转化为真实咨询效果。进一步研究需要加入随机错配命盘与 question-only 负对照，并在答案尚未公开的新命例上进行前瞻性、预注册评测。

这项工作的落点不是给命理贴上“科学”或“迷信”的标签，而是划清证据边界：什么已经被测量，什么仍然只是解释，什么尚未经过检验。一个可信的开放基准，应让每个分数都同时带着题集版本、覆盖率、置信区间、污染风险、运行条件和适用范围出现。这样做能让“准不准”不再只是产品自己的宣传语，而更接近一个可以被独立检查的问题。

## 附录 A：产品链路内部案例（事后证据）

如文首利益关系所述，作者运营 sjms.ai。将同一公开题集用于产品内部审计时，完整产品链路一度低于更简单的管线。现有记录支持两个值得后续验证的假说：

1. 在 200 题内部审计中，两个引擎共同答错 97 题，其中约 84% 选择了同一个错误选项。这个现象与共享输入或共享叙事影响一致，但也可能来自题目固有难度或模型共享先验，不能单独证明“上下文污染”。
2. 在一组 40 题的同构上下文消融中，保留主分析原文时为 27.5%，换成中性事实要点为 40.0%，完全移除历史为 42.5%。它提示结论性历史可能产生锚定，但该实验是开发后的事后分析，且使用已反复接触的公开题，不能当作独立泛化证据。

这类结果支持一个保守工程原则：优先注入可核验事实，避免把弱模型判断或既有叙事当成事实。然而，确认这一机制仍需要新题、预先锁定条件、完整运行日志和命主级区间。原始产品复盘叙述见公开论文草稿。
